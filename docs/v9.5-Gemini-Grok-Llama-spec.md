## 1. Rationale and Goals

v9.5-Gemini-first collapsed to a single vendor (Google) on the native `generateContent` endpoint to simplify the contest demo. That simplification cost us the strongest pitch element: **cross-vendor epistemic humility** — the live demo of three independent models agreeing (or disagreeing) on a flag is what makes a fact-check sidebar feel principled rather than oracular.

v9.5-Gemini-Grok-Llama brings consensus back, but on cheaper terms than v9.1:

- **Single endpoint schema** (Vertex AI OpenAI-compatible `chat/completions`) — handles all three vendors uniformly, so we don't re-create the per-vendor adapter mess of v9.1.
- **Conf-4/5 gating** — consensus only fires on high-confidence flags. Conf-2/3 cards stay single-model. Steady-state call cost is 1 model; the 3-model burst is rare and bounded.
- **Opt-in via Options toggle** — default off so first-run cost is identical to Gemini-first. The "epistemic humility" demo is one click away for the contest pitch but invisible to casual installs.
- **Data-driven model registry** — adding Claude as a fourth voice in v9.6 is a single row, not a refactor.

What this spec does NOT change vs. v9.5-Gemini-first: auth tracks, LM Studio local mode, the Gemini-first prompt re-tuning work, the chyron-on-conf-4/5 gating, PII-clean invariant, the iCloud zip recipe. Those carry forward.

## 4. Consensus Implementation

### Trigger Logic

```javascript
// In background.js, after primary Gemini classifier returns:
async function maybeFireConsensus(cue, primaryFlag) {
  const opts = await chrome.storage.local.get(["consensusEnabled", "voiceLlamaEnabled", "voiceGrokEnabled"]);
  if (!opts.consensusEnabled) return;
  if (primaryFlag.confidence < 4) return;

  const voices = [{ vendor: "google", verdict: primaryFlag.verdict }];
  const tasks = [];

  if (opts.voiceLlamaEnabled) {
    tasks.push(
      withTimeout(callVertex("consensus-llama", buildConsensusMessages(cue, primaryFlag)), 4000)
        .then(r => voices.push({ vendor: "meta", verdict: r.verdict }))
        .catch(() => { /* no-vote on timeout/error */ })
    );
  }
  if (opts.voiceGrokEnabled) {
    tasks.push(
      withTimeout(callVertex("consensus-grok", buildConsensusMessages(cue, primaryFlag)), 4000)
        .then(r => voices.push({ vendor: "xai", verdict: r.verdict }))
        .catch(() => { /* no-vote */ })
    );
  }

  await Promise.allSettled(tasks);

  const verdict = aggregateConsensus(voices);
  safeSend({ kind: "consensus", cue, verdict, voices });
}

function aggregateConsensus(voices) {
  // Need at least 2 voices that returned to make any call
  if (voices.length < 2) return "inconclusive";

  // Count verdicts. Agreement = 2-of-3 (or 2-of-2 if one timed out).
  const counts = {};
  voices.forEach(v => { counts[v.verdict] = (counts[v.verdict] || 0) + 1; });
  const maxCount = Math.max(...Object.values(counts));

  if (voices.length === 3 && maxCount >= 2) return "agree";
  if (voices.length === 2 && maxCount === 2) return "agree";
  // 3 voices all disagree, or 2 voices disagree
  return voices.length === 2 ? "inconclusive" : "split";
}
```

### Badge Rendering Rules

- `verdict === "agree"` → render consensus badge ("2 of 3 agree" or "Llama + Grok agree" — exact copy TBD).
- `verdict === "split"` → no badge. Gemini flag still renders normally.
- `verdict === "inconclusive"` → no badge. Treated identically to split for UI purposes.

The 2-of-2 case (only one consensus voice responded) is **explicitly treated as inconclusive** — we don't render an "agree" badge from a single corroborating voice because that's not consensus, it's a duo.

```javascript
const MODEL_REGISTRY = {

  "chitchat":          { slug: "google/gemini-2.5-flash", region: "global",   extraBody: GEMINI_NO_THINK },
  "citation":          { slug: "google/gemini-2.5-flash", region: "global",   extraBody: GEMINI_NO_THINK },
  "dossier":           { slug: "google/gemini-2.5-pro",   region: "global",   extraBody: {} },
  "consensus-llama":   { slug: "meta/llama-4-maverick-17b-128e-instruct-maas", region: "us-east5", extraBody: {} },
  "consensus-grok":    { slug: "xai/grok-4.1-fast-reasoning", region: "global", extraBody: {} },
  // v9.6 delta:
  // "consensus-claude": { slug: "anthropic/claude-haiku-...", region: "us-east5", extraBody: {} },
};

const GEMINI_NO_THINK = { google: { thinking_config: { thinking_budget: 0 } } };
// (VERIFY) exact key path. Google docs sometimes show `extra_body.thinking_config` (no `google.` wrapper)
// for the OpenAI-compat endpoint. Smoke-test both forms; record the working one in CLAUDE.md.

function buildUrl({ region }, projectId) {
  if (region === "global") {
    return `https://aiplatform.googleapis.com/v1beta1/projects/${projectId}/locations/global/endpoints/openapi/chat/completions`;
  }
  return `https://${region}-aiplatform.googleapis.com/v1beta1/projects/${projectId}/locations/${region}/endpoints/openapi/chat/completions`;
}

async function callVertex(role, messages, { maxTokens = 1024, temperature = 0.1 } = {}) {
  const entry = MODEL_REGISTRY[role];
  if (!entry) throw new Error(`Unknown role: ${role}`);
  const token = await getAccessToken(/* sa or chrome.identity */);
  const projectId = await getProjectId();

  const body = {
    model: entry.slug,
    messages,
    max_tokens: maxTokens,
    temperature,
  };
  if (entry.extraBody && Object.keys(entry.extraBody).length) {
    body.extra_body = entry.extraBody;
  }

  const resp = await fetch(buildUrl(entry, projectId), {
    method: "POST",
    headers: { Authorization: `Bearer ${token}`, "Content-Type": "application/json" },
    body: JSON.stringify(body),
  });
  if (!resp.ok) throw new Error(`Vertex ${role} ${resp.status}`);
  return await resp.json();
}
```

## 6. Cost Guardrails

### Grok's 664-token system wrapper

The Vertex/xAI gateway injects a system wrapper of ~664 tokens around every Grok request. This means every Grok consensus call has a ~660-token baseline cost before user content even reaches the model. Mitigations baked into this spec:

1. **Conf-4/5 gating** — Grok only fires on high-confidence flags. A typical 30-minute podcast clip produces maybe 2-5 conf-4/5 flags, capping Grok cost at 2-5 calls per session.
2. **Per-voice toggle** — Options-page checkbox lets the user disable Grok specifically while keeping Llama. Default off; the consensus toggle is the gross enable, per-voice toggles refine.
3. **Tooltip surfacing** — the Grok checkbox tooltip explicitly notes the ~660-token tax so the user makes an informed call.
4. **Documentation** — add a note in `CLAUDE.md` under a "Vendor gotchas" section (or create that section): "Grok via Vertex/xAI gateway has a ~664-token system prompt wrapper injected per call. Account for this in cost projections." This protects future-me from re-discovering it after a billing surprise.

### Llama and Gemini cost notes

- Llama 4 Maverick on Vertex MaaS — no reasoning overhead, predictable per-token cost. Region pinned to us-east5.
- Gemini Pro with thinking allowed burns ~188 reasoning tokens on trivial responses. Acceptable for pre-roll dossier (one call per video). **Do not enable Pro for any other role without first re-enabling `thinking_budget: 0`.**

## 7. Prompt Re-tuning Checklist

The Gemini-specific re-tuning from v9.5-Gemini-first (§4 of that spec) carries forward unchanged. Additions for the multi-vendor path:

- **OpenAI-compat structured output.** Native Gemini supports `responseMimeType: "application/json"` + `responseSchema`. **The OpenAI-compat endpoint may not expose these.** (VERIFY.) If not available, fall back to: (a) strong prompt instruction "Respond ONLY with valid JSON, no prose, no fences," (b) `response_format: { type: "json_object" }` if Vertex's OpenAI-compat surface supports that OpenAI-style field, (c) `format_guard.js` defensive parsing as a backstop.
- **Vendor-uniform consensus prompt.** The consensus-voice prompt must be neutral — same text goes to Gemini, Llama, and Grok. No vendor names in the prompt. Format: "Given this claim and context, return `{ verdict: 'true'|'false'|'misleading'|'unverifiable' }`. No prose."
- **Llama format-following.** Llama 4 Maverick tends to over-explain. Lead the consensus prompt with "Respond with a single JSON object. Do not explain your reasoning. Do not use markdown."
- **Grok reasoning leakage.** Grok 4.1 Fast Reasoning may emit reasoning even when not asked. The OpenAI-compat envelope should strip reasoning into a separate field, but format_guard should also defensively skip any text before the first `{` and after the last `}`.
- **Few-shot examples in consensus prompt.** 2-3 examples showing claim → JSON verdict. Required for Llama and Grok to hit format reliably.

## 9. Acceptance Criteria

v9.5-Gemini-Grok-Llama ships when ALL of v9.5-Gemini-first's acceptance criteria pass AND:

- With consensus toggle **off**, behavior is byte-for-byte equivalent to v9.5-Gemini-first (no Llama/Grok calls made; no consensus badges rendered; cost identical).
- With consensus toggle **on** and both Llama + Grok enabled: a conf-4/5 card on a known test clip (e.g., Murray/Smith dialogue with verifiable factual claims) fires both consensus calls within 100ms of the primary Gemini flag.
- Consensus badge renders when 2+ vendors agree on `verdict`.
- No badge renders when all three disagree (split) OR when only one voice responded (inconclusive).
- If either Llama or Grok times out at 4s, the card still renders the primary Gemini flag; consensus result is `inconclusive` (or `agree` if the surviving voice agrees with Gemini and we relax the 2-of-3 rule — **default behavior is strict: 2-of-3 only**).

- Adding a fourth voice (Claude) requires editing exactly one file: the model registry. Verified by code review.
